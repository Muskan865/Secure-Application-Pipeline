# Build & Break: Secure Application Pipeline

## Overview
This repository contains a DevSecOps project where we design, build, test, exploit, and secure a web application. The goal is to implement a complete secure software development lifecycle using both automated and manual security practices.

## Objectives
- Develop and containerize a web application
- Integrate a CI/CD pipeline with security checks
- Perform SAST, DAST, and SCA testing
- Identify and exploit vulnerabilities
- Apply fixes and validate through re-testing
- Document findings in professional reports

## Scope
- Application source code
- Docker setup and deployment configuration
- CI/CD pipeline implementation
- Threat model and attack surface analysis
- Security testing results (SAST, DAST, SCA)
- Exploitation report
- Remediation and re-test report
- Executive summary for non-technical audience

## Workflow
1. Design application and threat model  
2. Build and dockerize the application  
3. Integrate SAST and SCA into pipeline  
4. Perform DAST and manual penetration testing  
5. Exploit identified vulnerabilities  
6. Fix issues and re-test
